SelfSync · frontend design

A sync UI that stays quiet, reads at a glance, and looks like it shipped with Obsidian.

The settings tab, first-run wizard, and status affordances for the SelfSync plugin — rendered in Obsidian's own visual language. Every choice here traces to what the Obsidian community actually praises (and complains about) in plugin UI, and to Obsidian's official plugin guidelines.

The through-line from the research: trust comes from a single, unambiguous status you never have to hunt for — quiet in the status bar, loud only when it needs a decision.

Status-first

Sync users' one recurring question is “did my change make it?” The card answers it before anything else and pairs it with “last synced” — the trust signal the research kept surfacing.

Native, not bolted-on

Rows are Obsidian's setting-item; headings use setHeading(); every color is a token like --color-green, not a hex. Fixes our current raw <h3> + hardcoded #3fb950.

Progressive disclosure

Strong defaults mean most users never scroll past “What syncs.” Advanced collapses everything else — the pattern LiveSync's “power-user mode” is praised for.

Trust, stated plainly

The 🔒 line — “SelfSync's own login is never synced” — is shown whenever config sync is on, because it's the anxiety users voice about sync plugins.

One indicator, five unambiguous states

The community's sharpest lesson (LiveSync issue #572): a status icon that inverts expectation — 💤 meaning “done” — destroys trust, and a spinner that never resolves is worse than none. So SelfSync uses a distinct glyph + semantic color + label for each state (glyph carries it for colorblind users), mirrored on the status bar and the mobile-visible ribbon. Toggle states in the toolbar above; every surface updates together.

First-run wizard — not a settings dump

Opens automatically on first launch. Each step validates before the next unlocks (you can't advance past a server you can't reach). “I have a setup link” routes device #2 through the same reachability check — the Setup-URI idea LiveSync users specifically like, made safe.

Add another device

A selfsync:// link + scannable QR carrying the server and username — never the password. Bootstraps a second device without re-deriving config, the friction point sync users call out.
